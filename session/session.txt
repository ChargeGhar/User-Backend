Q1: When powerbank return flow triggered, is wallet/points
    deducted if amount is available?

A1: YES ✅
    - Auto-collection (_auto_collect_payment) runs immediately
    - Checks user balance (wallet + points)
    - If sufficient: deducts and marks PAID
    - If insufficient: marks PENDING, notifies user
    - Code: return_powerbank.py:117-164

Q2: When user pays via pay-due, how is amount_paid getting
    updated? Is it overriding or += happening?

A2: NEITHER - amount_paid is NOT MODIFIED ✅
    - amount_paid stays as calculated at return time
    - pay-due only changes payment_status: PENDING → PAID
    - pay-due only resets overdue_amount to 0
    - Code: rental_payment.py:192-200
    - update_fields = ['overdue_amount', 'payment_status', 'updated_at']
    - Notice: 'amount_paid' is NOT in update_fields

Q3: User has no balance except minimum required for postpaid
    rental, when powerbank return flow triggered, is wallet/
    points deducted or not?

A3: NOT DEDUCTED ✅
    - Auto-collection checks if balance >= required_due
    - If insufficient: skips deduction, marks PENDING
    - User must add balance and pay via pay-due endpoint
    - Verified in testing: wallet NPR 10, cost NPR 50
    - Result: status=PENDING, balance unchanged

════════════════════════════════════════════════════════════
  🎯 BUSINESS LOGIC SUMMARY
════════════════════════════════════════════════════════════

POSTPAID RENTAL:
  1. Start: No upfront charge, amount_paid = 0
  2. Return: Calculate cost, set amount_paid, try auto-collect
  3. Auto-collect success: Deduct immediately, mark PAID
  4. Auto-collect fail: Mark PENDING, user pays later
  5. Pay-due: Deduct from wallet/points, mark PAID

PREPAID RENTAL:
  1. Start: Charge upfront, amount_paid = package.price
  2. Return: No additional charge (unless late)
  3. Late return: Calculate overdue_amount, try auto-collect
  4. Cancel: Refund (full if free window, partial if late)

CANCEL FLOW:
  - Free window: 5 minutes (configurable)
  - Within window: Full refund (prepaid) or no charge (postpaid)
  - After window: Cancellation fee applies
  - Prepaid: Refund = amount_paid - fee
  - Postpaid: Charge = fee (deduct from wallet)

OVERDUE/LATE FEES:
  - Calculated using LateFeeService
  - Stored in overdue_amount field
  - Added to required_due for payment
  - Auto-collection attempts on return
